## 协议 Markdown 渲染修复（17:30）

### 问题
- `register.php` 用 `e($content)` + `white-space:pre-wrap`，Markdown 源码原样展示
- `submit.php` 直接 `echo $content`，Markdown 未渲染
- `admin/agreements.php` 预览用 JS `<pre>` 包裹原始文本

### 方案
采用 **前端 marked.js（CDN）** 渲染，PHP 端只传原始 Markdown 字符串给 JS。

### 修改
- **register.php**：弹窗 body 内容改为 `<div id="agreeModalBody">`，在 `showAgreementModal()` 调用时渲染；`white-space:pre-wrap` 改为 Markdown 样式；页面底部引入 `marked@9` CDN；Markdown 原文通过 `json_encode` 传给 JS 变量 `_regAgreementMd`
- **submit.php**：发布协议弹窗 body 改为 `id="publishAgreementBody"`，`showPublishAgreementModal()` 调用时渲染；页面底部引入 `marked@9` CDN
- **admin/agreements.php**：预览区改为 `data-md` 属性存储原文，`togglePreview()` 显示时调用 `marked.parse()` 渲染；引入 `marked@9` CDN
- Markdown 样式（h1-h6/p/ul/ol/li/strong/em/hr/code/pre/blockquote）在各页面内联 CSS 中定义
